Welcome to the webapp interface for the SEEKR (SEquence Evaluation through Kmer Representation) algorithm. SEEKR is a novel algorithm used to quantify similarities and differences in genomic sequences, particularly those of long non-coding RNAs (lncRNAs).

Input

SEEKR requires several inputs and options be set before running.


1) User fasta file

This file should contain the transcripts the user is interested in studying via SEEKR. The file must be in fasta format. See https://blast.ncbi.nlm.nih.gov for a definition of the fasta format. Additionally, we provide two sample fasta files for testing:

example2.fa
example15.fa

2) Comparison fasta file (default = User Set)

A second fasta file must be declared to compare against the transcripts of the user fasta file. There are three types of options for this file:

  1. User Set – This will compare the User fasta file to itself.
  2. Preloaded GENCODE fasta files – We provide the latest human and mouse lncRNA annotations from GENCODE.
  3. Upload Set – The user can choose to upload a second fasta file for comparison. An example use case for uploading a second fasta file would be using a zebrafish lncRNA annotation if the transcripts in the `User fasta file` are all from the zebrafish genome.

3) Normalization Set (default = All Human lncRNAs (GENCODE))

After counting kmers for each sequence, SEEKR normalizes each kmer by subtracting by the mean and dividing by the standard deviation of each kmer. In many cases, it is reasonable to substitute these mean and standard deviation arrays with other, related values.

  1. All Human lncRNAs (GENCODE) – A common use case is when comparing a relatively small (< 100) lncRNAs of interest to each other. Because the set is small, extreme values are more likely to arise. It is often of interest to measure how a small set of lncRNAs compares to themselves, but when normalized to a full annotation of lncRNAs.
  2. All Mouse lncRNAs (GENCODE) – This option is the same as above, but using mouse lncRNAs.
  3. User Set – If studying a larger set of lncRNAs, such as a zebrafish annotation, select the user set for normalization.

4) Kmer length (default = 1)

The size of the kmer to count in the fasta files. We recommend k=6 for most applications, though smaller kmers will process exponentially faster.



Output

Press the Submit button and the algorithms output will be visualized in the ‘Results’ tab (files container over 200 RNA sequences and Kmers above 6 will not produce visuals)


1) Sequence Kmer Profiles:

Displays the kmer count profiles for sequences in the User Set. Each sequence is a row on the y axis, and each kmer is a column on the x axis. The sequences are automatically arranged according to a hierarchical clustering on the kmer contents of each sequence. Clicking on either the row or column labels will re-order the graph based on the values in the row or column clicked.

2) Sequence Pearson Correlation Matrix

Both the x axis and the y axis represent sequences from the User fasta file. Each element in the matrix indicates the Pearson Correlation R value between the two sequences in the given row and column. The sequences are automatically arranged according to the same hierarchical clustering as the Sequence Kmer Profiles visual. Clicking on either the row or column labels will re-order the graph based on the values in the row or column clicked.
